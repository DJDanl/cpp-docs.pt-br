# <a name="styles-used-by-mfc"></a>Estilos usados pelo MFC

Use os seguintes sinalizadores de estilo para especificar a aparência e o comportamento da janela ou do controle quando criar o objeto MFC correspondente. Na maioria dos casos, esses estilos são definidos `Create` no parâmetro *dwStyle* da função classe.

## <a name="button-styles"></a><a name="button-styles"></a>Estilos de botão

Os estilos de botão aplicam-se a objetos [cbutton class,](../../mfc/reference/cbutton-class.md) como botões de rádio, caixas de seleção e botões. Especifique uma combinação de estilos no parâmetro *dwStyle* de [CButton::Create](../../mfc/reference/cbutton-class.md#create). Para obter mais informações sobre os estilos de botões no Windows, consulte [Estilos de botão (Windows)](/windows/win32/Controls/button-styles).

### <a name="button-types"></a>Tipos de botões

A tabela a seguir lista os tipos de botões. Você pode escolher opcionalmente um dos seguintes. Se você não especificar um tipo de botão, o padrão será BS_PUSHBUTTON.

|Type|Descrição|
|----------|-----------------|
|Bs_3state|Cria um botão de caixa de seleção com três estados: BST_CHECKED, BST_INDETERMINATE e BST_UNCHECKED. Clicar no botão envia uma notificação BN_CLICKED para a janela do proprietário, mas não altera o estado do botão. Por padrão, o texto associado é exibido à direita da caixa de seleção. Para exibir o texto à esquerda da caixa de seleção, use o estilo BS_LEFTTEXT ou BS_RIGHTBUTTON.|
|Bs_auto3state|Cria um botão de caixa de seleção com três estados: BST_CHECKED, BST_INDETERMINATE e BST_UNCHECKED. Clicar no botão envia uma notificação BN_CLICKED para a janela do proprietário e altera o estado do botão. O botão diz ciclo na ordem de BST_CHECKED, BST_INDETERMINATE e BST_UNCHECKED. Por padrão, o texto associado é exibido à direita da caixa de seleção. Para exibir o texto à esquerda da caixa de seleção, use o estilo BS_LEFTTEXT ou BS_RIGHTBUTTON.|
|BS_AUTOCHECKBOX|Cria um botão de caixa de seleção com dois estados: BST_CHECKED e BST_UNCHECKED. Clicar no botão envia uma notificação BN_CLICKED para a janela do proprietário e altera o estado do botão. Por padrão, o texto associado é exibido à direita da caixa de seleção. Para exibir o texto à esquerda da caixa de seleção, use o estilo BS_LEFTTEXT ou BS_RIGHTBUTTON.|
|Bs_autoradiobutton|Cria um botão de rádio com dois estados: BST_CHECKED e BST_UNCHECKED. Botões de rádio são geralmente usados em grupos, com cada grupo tendo um máximo de uma opção verificada por vez. Clicar no botão envia uma notificação BN_CLICKED para a janela do proprietário, define o estado do botão de rádio clicado para BST_CHECKED e define os estados de todos os outros botões de rádio no grupo de botões para BST_UNCHECKED. Por padrão, o texto associado é exibido à direita do botão de rádio. Para exibir o texto à esquerda do botão de rádio, use o estilo BS_LEFTTEXT ou BS_RIGHTBUTTON.|
|Bs_checkbox|Cria um botão de caixa de seleção com dois estados: BST_CHECKED e BST_UNCHECKED. Clicar no botão envia uma notificação BN_CLICKED para a janela do proprietário, mas não altera o estado do botão. Por padrão, o texto associado é exibido à direita da caixa de seleção. Para exibir o texto à esquerda da caixa de seleção, use o estilo BS_LEFTTEXT ou BS_RIGHTBUTTON.|
|BS_COMMANDLINK|Cria um botão de link de comando. Um botão de link de comando é um botão de comando específico do Windows Vista que exibe uma seta verde à esquerda do texto principal e uma nota abaixo do texto principal. Você pode definir o texto de nota usando [CButton::SetNote](../../mfc/reference/cbutton-class.md#setnote).|
|BS_DEFCOMMANDLINK|Cria um botão de link de comando. Um botão de link de comando é um botão de comando específico do Windows Vista que exibe uma seta verde à esquerda do texto principal e uma nota abaixo do texto principal. Você pode definir o texto de nota usando [CButton::SetNote](../../mfc/reference/cbutton-class.md#setnote). Se o botão estiver em uma caixa de diálogo, pressionar a tecla ENTER envia uma notificação BN_CLICKED para a caixa de diálogo mesmo quando o botão não tiver o foco de entrada.|
|BS_DEFPUSHBUTTON|Cria um botão de comando que tem uma borda preta pesada. Se o botão estiver em uma caixa de diálogo, pressionar a tecla ENTER envia uma notificação BN_CLICKED para a caixa de diálogo mesmo quando o botão não tiver o foco de entrada.|
|Bs_defsplitbutton|Cria um botão de divisão. Um botão split é um botão de comando específico do Windows Vista que contém um botão adjacente a uma seta para baixo. Quando você clica no botão, o comando padrão é executado. Quando você clica na seta suspenso, um menu de comandos adicionais é exibido. Se o botão de divisão estiver em uma caixa de diálogo, pressionar a tecla ENTER envia uma notificação BN_CLICKED para a caixa de diálogo mesmo quando o botão não tiver o foco de entrada|
|BS_GROUPBOX|Cria um retângulo no qual outros botões podem ser agrupados. O texto associado a este estilo é exibido no canto superior esquerdo do retângulo.|
|BS_OWNERDRAW|Cria um botão desenhado pelo proprietário. A estrutura `DrawItem` chama o método quando um aspecto visual do botão foi alterado. Este estilo deve ser definido `CBitmapButton` quando você usar a classe.|
|BS_PUSHBUTTON|Cria um botão de comando que envia uma notificação BN_CLICKED à janela do proprietário quando o usuário clica no botão.|
|Bs_radiobutton|Cria um botão de rádio com dois estados: BST_CHECKED e BST_UNCHECKED. Botões de rádio são geralmente usados em grupos, com cada grupo tendo um máximo de uma opção verificada por vez. Clicar no botão envia uma notificação BN_CLICKED para a janela do proprietário, mas não altera automaticamente o estado de qualquer botão no grupo. Por padrão, o texto associado é exibido à direita do botão de rádio. Para exibir o texto à esquerda do botão de rádio, use o estilo BS_LEFTTEXT ou BS_RIGHTBUTTON.|
|Bs_splitbutton|Cria um botão de divisão. Um botão split é um botão de comando específico do Windows Vista que contém um botão adjacente a uma seta para baixo. Quando você clica no botão, o comando padrão é executado. Quando você clica na seta suspenso, um menu de comandos adicionais é exibido.|
|BS_USERBUTTON|Obsoleto, mas fornecido para compatibilidade com versões de 16 bits do Windows. Os aplicativos baseados no Win32 devem usar BS_OWNERDRAW em vez disso.|

### <a name="radio-button-and-check-box-styles"></a>Botões de rádio e estilos de caixa de seleção

A tabela a seguir lista estilos específicos para botões de rádio e caixas de seleção. Esses estilos são ignorados em todos os outros tipos de botões. Você pode escolher opcionalmente um ou mais dos seguintes.

|Estilo|Descrição|
|-----------|-----------------|
|BS_LEFTTEXT|Quando combinado com um botão de rádio ou estilo caixa de seleção, o texto aparece no lado esquerdo do botão de rádio ou caixa de seleção.|
|BS_RIGHTBUTTON|Quando combinado com um botão de rádio ou estilo caixa de seleção, o texto aparece no lado esquerdo do botão de rádio ou caixa de seleção. Este estilo é idêntico ao estilo BS_LEFTTEXT.|
|BS_PUSHLIKE|Faz com que uma caixa de seleção ou botão de rádio pareça e se comporte como um botão de comando. O botão aparece pressionado quando seu estado é BST_CHECKED, pressionado e escurecido quando seu estado é BST_INDETERMINATE, e liberado quando seu estado é BST_UNCHECKED.|

### <a name="button-text-alignment-styles"></a>Estilos de alinhamento de texto de botão

A tabela a seguir lista opções de alinhamento de texto horizontal e vertical. Você pode escolher opcionalmente um dos seguintes.

|Estilo|Descrição|
|-----------|-----------------|
|BS_LEFT|À esquerda alinha o texto no retângulo do botão. No entanto, se o botão for uma caixa de seleção ou um botão de rádio que não tenha o estilo BS_RIGHTBUTTON, o texto fica alinhado no lado direito da caixa de seleção ou no botão de rádio.|
|BS_RIGHT|A direita alinha o texto no retângulo do botão. No entanto, se o botão for uma caixa de seleção ou um botão de rádio que não tenha o estilo BS_RIGHTBUTTON, o texto está alinhado no lado direito da caixa de seleção ou no botão de rádio.|
|BS_CENTER|Centraliza o texto horizontalmente no retângulo do botão.|
|BS_TOP|Coloca o texto na parte superior do retângulo do botão.|
|BS_BOTTOM|Coloca o texto na parte inferior do retângulo do botão.|
|BS_VCENTER|Centraliza o texto verticalmente no retângulo do botão.|

### <a name="button-content-options"></a>Opções de conteúdo de botão

A tabela a seguir lista opções que indicam o que é exibido no botão. Tipos de botões que só exibem texto ignoram esses estilos. Você pode escolher opcionalmente um dos seguintes.

|Estilo|Descrição|
|-----------|-----------------|
|Bs_bitmap|Especifica que o botão exibe um bitmap.|
|BS_ICON|Especifica que o botão exibe um ícone.|
|BS_TEXT|Especifica que o botão exibe texto.|

### <a name="other-button-options"></a>Outras opções de botões

A tabela a seguir lista opções adicionais que você pode usar com qualquer tipo de botão. Você pode escolher opcionalmente um ou mais dos seguintes.

|Estilo|Descrição|
|-----------|-----------------|
|BS_FLAT|Especifica que o botão é bidimensional e não é desenhado com sombreamento padrão para criar uma imagem tridimensional.|
|BS_MULTILINE|Envolve o texto do botão em várias linhas se a seqüência de texto for muito longa para caber em uma única linha no retângulo do botão.|
|BS_NOTIFY|Habilita um botão para enviar mensagens de notificação BN_DBLCLK, BN_KILLFOCUS e BN_SETFOCUS para sua janela pai. Observe que os botões enviam a notificação BN_CLICKED, independentemente de este estilo ser especificado.|

## <a name="combo-box-styles"></a><a name="combo-box-styles"></a>Estilos de caixa de combo

Os seguintes estilos de caixa de combinação estão disponíveis no MFC. Para obter mais informações sobre os estilos de caixa de combinação no Windows, consulte [Combo Box Styles (Windows)](/windows/win32/Controls/combo-box-styles).

|Estilo|Descrição|
|-----------|-----------------|
|CBS_AUTOHSCROLL|Rola automaticamente o texto no controle de edição para a direita quando o usuário digita um caractere no final da linha. Se este estilo não for definido, somente o texto que se encaixa dentro do limite retangular é permitido.|
|CBS_DISABLENOSCROLL|A caixa de lista mostra uma barra de rolagem vertical desativada quando a caixa de lista não contém itens suficientes para rolar. Sem esse estilo, a barra de rolagem fica oculta quando a caixa de lista não contém itens suficientes.|
|Cbs_dropdown|Semelhante ao CBS_SIMPLE, exceto que a caixa de lista não é exibida a menos que o usuário selecione um ícone ao lado do controle de edição.|
|Cbs_dropdownlist|Semelhante ao CBS_DROPDOWN, exceto que o controle de edição é substituído por um item de texto estático que exibe a seleção atual na caixa de lista.|
|Cbs_hasstrings|Uma caixa de combinação de desenho do proprietário contém itens que consistem em strings. A caixa de combinação mantém a memória e os ponteiros `GetText` para as strings para que o aplicativo possa usar a função membro para recuperar o texto de um determinado item.|
|CBS_LOWERCASE|Converte-se em minúsculas todos os textos tanto na área de seleção quanto na lista.|
|CBS_NOINTEGRALHEIGHT|Especifica que o tamanho da caixa de combinação é exatamente o tamanho especificado pelo aplicativo quando criou a caixa de combinação. Normalmente, o Windows dimensiona uma caixa de combinação para que a caixa de combinação não exiba itens parciais.|
|CBS_OEMCONVERT|O texto inserido no controle de edição da caixa de combinação é convertido do caractere ANSI definido para o conjunto de caracteres OEM e, em seguida, de volta ao ANSI. Isso garante a conversão adequada de `AnsiToOem` caracteres quando o aplicativo chama a função Windows para converter uma seqüência ANSI na caixa de combinação para caracteres OEM. Este estilo é mais útil para caixas de combinação que contêm nomes de arquivos e se aplica apenas a caixas de combinação criadas com os estilos CBS_SIMPLE ou CBS_DROPDOWN.|
|Cbs_ownerdrawfixed|O proprietário da caixa de lista é responsável por desenhar seu conteúdo; os itens na caixa de lista são todos da mesma altura.|
|Cbs_ownerdrawvariable|O proprietário da caixa de lista é responsável por desenhar seu conteúdo; os itens na caixa de lista são variáveis em altura.|
|Cbs_simple|A caixa de lista é exibida o tempo todo. A seleção atual na caixa de lista é exibida no controle de edição.|
|Cbs_sort|Classifica automaticamente as strings inseridas na caixa de lista.|
|CBS_UPPERCASE|Converte-se em maiúsculas todos os textos tanto na área de seleção quanto na lista.|

## <a name="edit-styles"></a><a name="edit-styles"></a>Editar estilos

Os estilos de edição aplicam-se a objetos [cedit class.](../../mfc/reference/cedit-class.md) Especifique uma combinação de estilos no parâmetro *dwStyle* de [CEdit::Create](../../mfc/reference/cedit-class.md#create). Para obter mais informações sobre estilos de controle de edição no Windows, consulte [Editar estilos de controle (Windows)](/windows/win32/Controls/edit-control-styles).

|Estilo|Descrição|
|-----------|-----------------|
|Es_autohscroll|Rola automaticamente o texto para a direita por 10 caracteres quando o usuário digita um caractere no final da linha. Quando o usuário pressiona a tecla ENTER, o controle rola todo o texto de volta para a posição 0.|
|Es_autovscroll|Rola automaticamente o texto para uma página quando o usuário pressiona ENTER na última linha.|
|ES_CENTER|Centraliza o texto em um controle de edição de linha única ou multilinha.|
|ES_LEFT|Texto alinhado à esquerda em um controle de edição de linha única ou multilinha.|
|ES_LOWERCASE|Converte todos os caracteres em minúsculas à medida que são digitados no controle de edição.|
|ES_MULTILINE|Designa um controle de edição de várias linhas. (O padrão é linha única.) Se o estilo ES_AUTOVSCROLL for especificado, o controle de edição mostrará o máximo de linhas possível e rolará verticalmente quando o usuário pressionar a tecla ENTER. Se ES_AUTOVSCROLL não for dado, o controle de edição mostrará o máximo de linhas possível e emite bipes se ENTER for pressionado quando não for exibida mais linhas. Se o estilo ES_AUTOHSCROLL for especificado, o controle de edição de várias linhas rola automaticamente horizontalmente quando o caret passa pela borda direita do controle. Para iniciar uma nova linha, o usuário deve pressionar ENTER. Se ES_AUTOHSCROLL não for dado, o controle envolve automaticamente as palavras para o início da próxima linha quando necessário; uma nova linha também é iniciada se ENTER for pressionado. A posição do wordwrap é determinada pelo tamanho da janela. Se o tamanho da janela mudar, a posição wordwrap será mudada e o texto será reexibido. Os controles de edição de várias linhas podem ter barras de rolagem. Um controle de edição com barras de rolagem processa suas próprias mensagens de barra de rolagem. Editar controles sem rolar barras de rolagem como descrito acima e processar quaisquer mensagens de rolagem enviadas pela janela pai.|
|Es_nohidesel|Normalmente, um controle de edição oculta a seleção quando o controle perde o foco de entrada e inverte a seleção quando o controle recebe o foco de entrada. Especificar ES_NOHIDESEL exclui essa ação padrão.|
|ES_NUMBER|Permite que apenas dígitos sejam inseridos no controle de edição.|
|ES_OEMCONVERT|O texto inserido no controle de edição é convertido do caractere ANSI definido para o conjunto de caracteres OEM e, em seguida, de volta ao ANSI. ## <a name="extended-window-styles"></a><a name="extended-window-styles"></a>Estilos de janela estendidos

Estilos de janela estendidos aplicam-se a objetos [da Classe CWnd.](../../mfc/reference/cwnd-class.md) Especifique uma combinação de estilos no parâmetro *dwExStyle* de [CWnd::CreateEx](../../mfc/reference/cwnd-class.md#createex). Para obter mais informações sobre estilos de janela estendidos no Windows, consulte ['Estilos de janela estendidos' (Windows)](/windows/win32/winmsg/extended-window-styles).

|Estilo|Descrição|
|-----------|-----------------|
|WS_EX_ACCEPTFILES|Especifica que uma janela criada com esse estilo aceita arquivos de arrastar e soltar.|
|WS_EX_APPWINDOW|Força uma janela de alto nível na barra de tarefas quando a janela estiver visível.|
|Ws_ex_clientedge|Especifica que uma janela tem um visual 3D — ou seja, uma borda com uma borda afundada.|
|Ws_ex_contexthelp|Inclui um ponto de interrogação na barra de título da janela. Quando o usuário clica no ponto de interrogação, o cursor muda para um ponto de interrogação com um ponteiro. Se o usuário clicar em uma janela de criança, a criança receberá uma mensagem WM_HELP.|
|WS_EX_CONTROLPARENT|Permite que o usuário navegue entre as janelas da janela usando a tecla TAB.|
|WS_EX_DLGMODALFRAME|Designa uma janela com uma borda dupla que pode (opcionalmente) ser criada com uma barra de título quando você especifica rastas do WS_CAPTION sinalizador de estilo no parâmetro *dwStyle.*|
|WS_EX_LAYERED|A janela é uma [janela em camadas.](/windows/win32/winmsg/window-features) Este estilo não pode ser usado se a janela tiver um estilo de [classe](/windows/win32/winmsg/about-window-classes) de CS_OWNDC ou CS_CLASSDC. No entanto, o Microsoft Windows 8 suporta o estilo WS_EX_LAYERED para janelas infantis, onde versões anteriores do Windows suportam apenas para janelas de nível superior.|
|WS_EX_LEFT|Dá propriedades genéricas de janela alinhadas à esquerda. Esse é o padrão.|
|WS_EX_LEFTSCROLLBAR|Coloca uma barra de rolagem vertical à esquerda da área do cliente.|
|WS_EX_LTRREADING|Exibe o texto da janela usando propriedades de ordem de leitura da esquerda para a direita. Esse é o padrão.|
|WS_EX_MDICHILD|Cria uma janela de criança MDI.|
|WS_EX_NOPARENTNOTIFY|Especifica que uma janela filho criada com esse estilo não enviará a mensagem WM_PARENTNOTIFY para a janela dos pais quando a janela do filho for criada ou destruída.|
|WS_EX_OVERLAPPEDWINDOW|Combina os estilos WS_EX_CLIENTEDGE e WS_EX_WINDOWEDGE|
|WS_EX_PALETTEWINDOW|Combina os estilos WS_EX_WINDOWEDGE e WS_EX_TOPMOST.|
|WS_EX_RIGHT|Dá uma janela propriedades genéricas alinhadas à direita. Isso depende da aula de janelas.|
|WS_EX_RIGHTSCROLLBAR|Coloca uma barra de rolagem vertical (se presente) à direita da área do cliente. Esse é o padrão.|
|Ws_ex_rtlreading|Exibe o texto da janela usando propriedades de ordem de leitura da direita para a esquerda.|
|WS_EX_STATICEDGE|Cria uma janela com um estilo de borda tridimensional destinado a ser usado para itens que não aceitam a entrada do usuário.|
|Ws_ex_toolwindow|Cria uma janela de ferramenta, que é uma janela destinada a ser usada como uma barra de ferramentas flutuante. Uma janela de ferramenta tem uma barra de título mais curta que uma barra de título normal, e o título da janela é desenhado usando uma fonte menor. Uma janela de ferramenta não aparece na barra de tarefas ou na janela que aparece quando o usuário pressiona ALT+TAB.|
|Ws_ex_topmost|Especifica que uma janela criada com esse estilo deve ser colocada acima de todas as janelas não topo e permanecer acima delas mesmo quando a janela estiver desativada. Um aplicativo pode `SetWindowPos` usar a função membro para adicionar ou remover este atributo.|
|WS_EX_TRANSPARENT|Especifica que uma janela criada com este estilo deve ser transparente. Ou seja, todas as janelas que estão abaixo da janela não são obscurecidas pela janela. Uma janela criada com esse estilo recebe mensagens WM_PAINT somente depois de todas as janelas de irmãos abaixo dela terem sido atualizadas.|
|WS_EX_WINDOWEDGE|Especifica que uma janela tem uma borda com uma borda elevada.|
