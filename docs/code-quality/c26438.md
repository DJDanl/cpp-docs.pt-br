# <a name="c26438-no_goto"></a>C26438 NO_GOTO

"Evite". `goto`

**C++ Diretrizes principais**:\
ES.76: Evite irto

O uso `goto` de é amplamente considerado uma prática perigosa e propensa a erros. É aceitável apenas em código gerado, como em um analisador gerado a partir de uma gramática. Com recursos e utilitários C++ modernos fornecidos pela Biblioteca de `goto` Suporte de Diretrizes, deve ser fácil evitar completamente.

## <a name="remarks"></a>Comentários

- Esta regra adverte sobre `goto`qualquer ocorrência de , mesmo que aconteça em código morto, exceto código de modelo que nunca é usado e, conseqüentemente, ignorado pelo compilador.
- Os avisos podem se `goto`multiplicar quando uma macro contém . Os mecanismos de emissão de relatórios atuais apontam para todos os casos em que tal macro é expandida. Muitas vezes pode ser fixado em um lugar alterando a macro, ou evitando seu uso em favor de mecanismos mais mantenedores.

## <a name="example"></a>Exemplo

'goto clean-up' em macro

```cpp
#define ENSURE(E, L) if (!(E)) goto L;

void poll(connection &c)
{
    ENSURE(c.open(), end);                  // C26438

    while (c.wait())
    {
        connection::header h{};
        connection::signature s{};
        ENSURE(c.read_header(h), end);      // C26438
        ENSURE(c.read_signature(s), end);   // C26438
        // ...
    }

end:
    c.close();
}
```

'goto clean-up' em macro - substituído por gsl::finally

```cpp
void poll(connection &c)
{
    auto end = gsl::finally([&c] { c.close(); });

    if (!c.open())
        return;

    while (c.wait())
    {
        connection::header h{};
        connection::signature s{};
        if(!c.read_header(h))
            return;
       if(!c.read_signature(s))
            return;
        // ...
    }
}
```
